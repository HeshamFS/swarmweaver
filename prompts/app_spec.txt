<project_specification>
  <project_name>EUActPilot - EU AI Act Autonomous Compliance Engine</project_name>

  <overview>
    Build an autonomous AI compliance engine designed to help organizations achieve and maintain compliance with 
    the EU AI Act (Regulation 2024/1689). The platform implements a Marathon Agent architecture using Google's 
    Gemini 3 API and Agent Development Kit (ADK), providing continuous automated compliance monitoring, 
    documentation generation, risk classification, and regulatory update tracking.

    Core philosophy: COMPLIANCE-FIRST, EVIDENCE-BASED. Every classification requires multi-agent voting consensus,
    every documentation section must be complete before export, and every regulatory update triggers impact assessment.

    The platform features specialized agents (Discovery, Classification, Documentation, Monitoring, Regulatory),
    integrates with the AG-UI Protocol for real-time frontend-agent communication via CopilotKit, and supports
    the A2A Protocol for agent interoperability. The UI should be professional, compliance-focused, and optimized
    for enterprise workflows with clear visualization of multi-agent operations.

    Target deadline: August 2, 2026 (EU AI Act full application date)
    Penalties for non-compliance: Up to €35 million or 7% of global annual turnover
  </overview>

  <key_interactions>
    <classification_workflow>
      1. User selects AI system or enters description
      2. User clicks "Classify" or uses agent chat: "Classify this system"
      3. AG-UI workflow starts with RunStarted event
      4. Multi-agent workflow visible in UI:
         a. Legal Agent analyzes regulatory text (StepStarted/StepFinished)
         b. Technical Agent evaluates capabilities
         c. Contextual Agent assesses deployment context
         d. Consensus Agent builds voting result
      5. Agent responses stream via TextMessageContent events
      6. Tool calls shown with ToolCallStart/ToolCallArgs/ToolCallEnd
      7. State updates via StateDelta events sync UI
      8. Classification result displayed with voting breakdown
      9. If confidence < 0.85: Human review flag shown
      10. User can override with documented justification
      11. Result saved to database, audit log updated
    </classification_workflow>

    <documentation_generation>
      1. User navigates to system documentation page
      2. User clicks "Generate Documentation" or uses agent chat
      3. Agent workflow starts, progress shown in stepper
      4. Each Annex IV section generated sequentially:
         - Agent streams section content via AG-UI
         - UI updates section in real-time
         - Gaps identified and highlighted
      5. User reviews generated content
      6. User can edit sections manually
      7. User can request regeneration for specific sections
      8. Completion percentage updates automatically
      9. When 100% complete, Export button enabled
      10. User exports to PDF/DOCX/JSON
    </documentation_generation>

    <agent_chat_interaction>
      1. User opens agent chat (CopilotKit sidebar/popup)
      2. User types natural language request
      3. Request sent to AG-UI endpoint via CopilotKit
      4. Agent processes request with appropriate specialized agents
      5. Response streams back via AG-UI events:
         - TextMessageStart initiates message
         - TextMessageContent streams text chunks
         - ToolCallStart/Args/End for tool usage
         - StateDelta updates application state
         - TextMessageEnd completes message
      6. UI displays streaming response
      7. Tool calls rendered with expandable details
      8. State changes reflected in dashboard/views
      9. Human-in-the-loop: Agent can request confirmation
      10. User can stop generation, retry, or provide feedback
    </agent_chat_interaction>

    <regulatory_update_handling>
      1. Regulatory Monitor Agent detects new update
      2. Alert created with severity level
      3. Notification appears in dashboard
      4. User clicks to view update details
      5. Impact assessment shows affected systems
      6. Recommended actions listed
      7. User can ask agent for interpretation
      8. User acknowledges update
      9. Affected systems flagged for review
      10. Follow-up tasks created if needed
    </regulatory_update_handling>

    <compliance_monitoring>
      1. Monitoring Agent runs on schedule (daily/weekly)
      2. Agent checks for:
         - Model performance degradation
         - Data drift
         - Usage pattern changes
         - Documentation staleness
      3. Issues detected create alerts
      4. Alerts appear in dashboard and system views
      5. User investigates alert details
      6. User takes remediation action
      7. User resolves alert with notes
      8. Audit log records resolution
